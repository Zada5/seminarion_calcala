# Political Advertising Spend and Major Events in Israel (2020-2025)

This repository contains the data pipeline, analysis scripts, and generated outputs for a university economics seminar paper on political advertising spend in Israel. The project studies how weekly paid political advertising activity on Meta and Google relates to major political and security events over time.

The repository is designed to be auditable: raw exports are kept separate from cleaned inputs, cleaning stages are documented, placebo-event construction is reproducible, and the final tables used in the paper can be traced back to script-generated source files.

## Research Question

How strongly is weekly political advertising spend associated with major political and security events in Israel?

The analysis uses a weekly panel of advertising spend, event-study regressions, difference-in-differences style models, placebo-event checks, and descriptive summaries by platform and entity type.

## Data Sources

- **Meta Ads Library exports**: raw advertiser-level CSV files downloaded separately for each entity.
- **Google Ads political spend data**: workbook-based source data normalized into the same weekly schema as Meta.
- **Event timeline**: consolidated list of political and security incidents for the 2020-2025 seminar window.
- **Placebo event weeks**: generated by script using a fixed seed and validated distance rules from real events.

The preferred analysis inputs are the second-cleaning weekly files in `data/processed/second_cleaning/`. The full file lineage is documented in [`DATA_PIPELINE.md`](DATA_PIPELINE.md).

## Reproducibility

The project separates source data, cleaning stages, analysis scripts, and final outputs:

```text
raw Meta / Google exports
        -> first weekly aggregation
        -> manual cleaning and harmonization
        -> second-cleaning weekly panel
        -> descriptive summaries, event-study models, DiD models, placebo checks
        -> paper-ready tables and figures
```

Core commands from the repository root:

```bash
python3 scripts/03_generate_placebo_events.py
Rscript scripts/04_summarize_platform_publisher_spend.R
Rscript scripts/05_event_study.R
Rscript scripts/06_did.R
```

Python dependencies for the conversion scripts are listed in [`requirements.txt`](requirements.txt). The R scripts load their required packages directly.

## Methodological Notes

- The main seminar window uses Sunday-start weeks from `2020-01-05` through `2025-12-28`.
- The analysis unit is weekly advertising spend by entity and platform.
- Spend is analyzed in ILS.
- Regression scripts use `log(weekly_spend_ils)` as the dependent variable; rows with non-positive spend are excluded before estimation because `log(0)` is undefined.
- Event-study and DiD outputs use stacked event-window panels around real or placebo events.
- Placebo dates are generated once in `data/generated/placebo_events_2020_2025.csv` and validated by the analysis scripts.
- The repository preserves a documented `gamma_t` demonstration showing why calendar-week fixed effects are not used in the stacked specifications for this design.
